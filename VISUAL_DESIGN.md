# T&K Photo 视觉设计方案

> 艺术总监 / 前端视觉设计规范  
> 目标：线上摄影展，而非网盘相册  
> 风格：画廊、杂志、档案感、克制留白

---

## 一、整体视觉关键词

**核心关键词：** 留白 · 克制 · 质感 · 叙事

**氛围关键词：** 沉静 · 呼吸感 · 时间感 · 档案 · 展厅 · 手工感

### 设计哲学

这不是一个"相册"，而是一个**线上展厅**。每一页都应该像翻开一本摄影画册或走进一间小型画廊。

- **留白是主角**：照片之间的空白不是"没内容"，而是让眼睛休息、让照片呼吸的空间。
- **克制是品质**：去掉所有不必要的装饰——没有圆角、没有阴影、没有彩色背景、没有花哨动效。让照片本身说话。
- **质感靠细节**：衬线体的标题、等宽字体的参数、精确的间距、温和的过渡，这些组合起来传递"认真"和"专业"。
- **叙事靠文字**：每张照片有标题和一段描述，不是冷冰冰的文件名。文字是照片的延伸。

### 情绪板参考

| 参考对象 | 借鉴点 |
|---------|--------|
| Kinfolk 杂志 | 极致留白、图文关系、沉静气质 |
| 1x.com | 摄影展览感、大图展示、黑白灰配色 |
| Cereal Magazine | 排版节奏、衬线体运用、页面呼吸感 |
| 柏林 C/O 画廊网站 | 展览式布局、档案感、信息层级 |
| 日本美术馆网站 | 克制、米白背景、时间感 |

## 二、浅色主题配色

**主色调：暖灰米白系**，营造纸质画册的温润感。

| 色名 | 色值 | 用途 | 使用场景 |
|-----|------|------|---------|
| **背景主色** | `#FAFAF8` | 页面全局背景 | 整个网站的底色，偏暖的米白，不是纯白 |
| **卡片背景** | `#F2F0EC` | 卡片、面板底色 | AlbumCard 悬停态、EXIF 面板底色 |
| **主文字** | `#1A1A1A` | 标题、正文 | 大标题、照片标题、主要文字内容 |
| **辅助文字** | `#6B6B6B` | 副标题、描述 | 相册副标题、照片描述、页脚信息 |
| **弱化文字** | `#A0A0A0` | 时间、地点、标签 | EXIF 参数标签（如"相机""镜头"）、日期 |
| **点缀色** | `#8B7355` | 链接、悬停、强调 | 悬停时的文字色、当前导航指示、链接下划线 |
| **边框/分割线** | `#E8E4DE` | 分隔线、表单边框 | 极淡的分隔，几乎看不见但存在 |

### 配色使用原则

1. **背景只用 `#FAFAF8`**：不要给不同区块加不同背景色，整个网站底色统一，靠留白区分区块。
2. **文字层次靠透明度/色值**：不用彩色文字区分层级，用 `#1A1A1A` → `#6B6B6B` → `#A0A0A0` 三级递减。
3. **点缀色极其克制**：`#8B7355` 只用于交互状态（悬停、当前导航），不要大面积使用。
4. **照片是唯一的色彩**：在暖灰白背景上，照片本身的色彩就是视觉焦点。

## 三、深色主题配色

**主色调：炭黑暖灰系**，营造暗房/夜间展厅的氛围。

| 色名 | 色值 | 用途 | 使用场景 |
|-----|------|------|---------|
| **背景主色** | `#1A1A1A` | 页面全局背景 | 深色模式底色，不是纯黑，是带暖调的深炭灰 |
| **卡片背景** | `#242424` | 卡片、面板底色 | AlbumCard 悬停态、EXIF 面板底色 |
| **主文字** | `#F0EDE8` | 标题、正文 | 大标题、照片标题，偏暖的米白 |
| **辅助文字** | `#9A9A9A` | 副标题、描述 | 相册副标题、照片描述 |
| **弱化文字** | `#666666` | 时间、地点、标签 | EXIF 参数标签 |
| **点缀色** | `#C4A87A` | 链接、悬停、强调 | 悬停文字色、当前导航，比浅色模式的棕色更亮更暖 |
| **边框/分割线** | `#333333` | 分隔线 | 极淡的分隔线 |

### 深色模式设计原则

1. **不是简单的反色**：背景 `#1A1A1A` 不是纯黑（`#000000`），纯黑会太刺眼且没有质感。用带暖调的深炭灰。
2. **文字不用纯白**：`#F0EDE8` 是偏暖的米白，比纯白 `#FFFFFF` 更柔和，在深色背景上阅读更舒适。
3. **照片在深色模式下更突出**：深色背景天然是照片的最佳画框。
4. **点缀色提亮**：`#C4A87A` 比浅色模式的 `#8B7355` 更亮，在深色背景上有足够的对比度。

## 四、字体方案

### 字体族定义

| 用途 | 字体 | 来源 | 字重 | 说明 |
|-----|------|------|------|------|
| **英文标题** | Playfair Display | Google Fonts | 400 (Regular), 600 (SemiBold), 700 (Bold) | 高对比度衬线体，优雅、经典、有杂志感 |
| **中文标题** | Noto Serif SC (思源宋体) | Google Fonts | 400, 600, 700 | 衬线体，文艺感、书卷气、时间感 |
| **正文** | 系统字体栈 | 本地 | 400, 500 | `-apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif` |
| **EXIF 参数** | JetBrains Mono | Google Fonts | 400, 500 | 等宽字体，科技感、精确感、档案感 |

### 字号层级系统

| 层级 | 用途 | 桌面端字号 | 移动端字号 | 字重 | 行高 | 字间距 | 字体 |
|-----|------|-----------|-----------|------|------|--------|------|
| **Display** | 首页 Hero 大标题 | 64-80px | 40-48px | 400 | 1.1 | -0.02em | Playfair Display + Noto Serif SC |
| **H1** | 页面主标题 | 42-48px | 28-32px | 600 | 1.2 | -0.01em | Playfair Display + Noto Serif SC |
| **H2** | 相册标题 | 28-32px | 22-24px | 600 | 1.3 | 0 | Playfair Display + Noto Serif SC |
| **H3** | 照片标题 | 20-24px | 18-20px | 600 | 1.4 | 0 | Noto Serif SC |
| **Body** | 正文、描述 | 16px | 15px | 400 | 1.8 | 0.01em | 系统字体栈 |
| **Caption** | 照片说明、日期 | 13-14px | 12-13px | 400 | 1.6 | 0.02em | 系统字体栈 |
| **Label** | EXIF 标签 | 11-12px | 11px | 400 | 1.5 | 0.05em | 系统字体栈，全大写英文 |
| **Mono** | EXIF 数值 | 13-14px | 12-13px | 400 | 1.5 | 0 | JetBrains Mono |
| **Nav** | 导航链接 | 14px | 14px | 500 | 1 | 0.02em | 系统字体栈 |

### 字体使用规则

1. **大标题用衬线体**："T & K Photo"、相册标题、页面标题，全部用 Playfair Display（英文）+ Noto Serif SC（中文）。衬线体传递品质感和时间感。
2. **正文用系统无衬线体**：阅读长文本时无衬线体更清晰。用系统字体栈保证加载速度和跨平台一致性。
3. **EXIF 参数用等宽字体**：相机型号、光圈、快门等用 JetBrains Mono，营造档案感和精确感。
4. **标签全大写**：EXIF 参数的标签（如 "CAMERA"、"APERTURE"）用全大写英文，字号 11-12px，字间距加宽（0.05em），颜色用弱化文字色。
5. **中英文混排**：中文标题中夹杂英文时，英文也用 Playfair Display，但注意中英文基线对齐问题（Tailwind 中可用 `leading-tight` 控制）。
6. **不要加粗正文**：正文永远用 400 字重，靠字号和颜色区分层级，不靠粗细。

## 五、首页布局设计

### 页面结构（从上至下）

```
┌─────────────────────────────────────────────┐
│  Header (固定, 半透明背景)                    │
├─────────────────────────────────────────────┤
│                                             │
│                                             │
│           Hero 全屏区域                      │
│     背景图 + 暗色遮罩 + 居中标题              │
│                                             │
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│           精选相册区域                       │
│     AlbumCard 错落排列（2-4个卡片）           │
│                                             │
├─────────────────────────────────────────────┤
│  Footer (极简, 居中)                         │
└─────────────────────────────────────────────┘
```

### Hero 区域设计

**尺寸与定位：**
- 占满首屏：`100vh`，`width: 100%`
- 内容垂直水平居中

**背景处理：**
- 背景图：一张精选摄影大图，`object-fit: cover`，占满整个 Hero
- 暗色遮罩：`rgba(0, 0, 0, 0.35)` 的半透明黑色遮罩，确保白色文字可读
- 遮罩不是纯色：可以用轻微渐变，从底部 `rgba(0,0,0,0.5)` 到顶部 `rgba(0,0,0,0.2)`，让底部文字更清晰

**文字内容（居中）：**
- 主标题："T & K Photo" —— Playfair Display，64-80px（桌面）/ 40-48px（移动），字重 400，颜色 `#FFFFFF`
- 副标题："我们的摄影展厅" —— Noto Serif SC，18-20px，字重 400，颜色 `rgba(255,255,255,0.85)`
- 标题与副标题间距：16-20px

**装饰元素：**
- 不要按钮：Hero 区域不要"进入"按钮，让用户自然向下滚动
- 可以有一个极淡的向下滚动指示：底部居中，一个细线箭头，颜色 `rgba(255,255,255,0.4)`，2px 宽，缓慢上下浮动动画（振幅 8px，周期 2s）

**动效：**
- 页面加载时：背景图从 scale 1.05 缓慢缩放到 1.0（持续 8s，ease-out），营造极缓慢的"镜头推进"感
- 标题加载时：从 opacity 0, y 30px 渐变到 opacity 1, y 0（持续 0.8s，delay 0.3s）
- 副标题延迟 0.5s 淡入

### 精选相册区域设计

**顶部留白：**
- Hero 与相册区域之间：至少 120px 留白（桌面），80px（移动）

**区域标题（可选）：**
- 可以不设区域标题，让相册直接出现
- 如果要加："精选相册" 或 "Selected Works" —— H2 字号，居左对齐，颜色辅助文字色，下方 60px 留白

**AlbumGrid 错落布局：**

```
桌面端布局（max-width: 1200px，居中）：

┌─────────────────────────────────────────────┐
│            AlbumCard (全宽, span 2)          │
│            比例: 16:9 或 3:2                 │
├────────────────────────┬────────────────────┤
│   AlbumCard (半宽)      │   AlbumCard (半宽)  │
│   比例: 4:3 或 3:2      │   比例: 4:3 或 3:2  │
├────────────────────────┴────────────────────┤
│            AlbumCard (全宽, span 2)          │
│            比例: 16:9 或 3:2                 │
└─────────────────────────────────────────────┘

间距：
- 卡片之间 gap: 40px（桌面），24px（移动）
- 卡片与页面边缘 padding: 48px（桌面），24px（移动）
```

**布局规则：**
- 第 1 个卡片：全宽（`grid-column: span 2`），高度自适应图片比例
- 第 2、3 个卡片：半宽并排（各 `grid-column: span 1`）
- 第 4 个卡片：全宽
- 如卡片数量不足，按此规律循环
- 移动端（< 768px）：全部单列，全宽

## 六、相册卡片设计

### AlbumCard 结构

```
┌─────────────────────────────────────────┐
│                                         │
│           封面图 (全宽, 原比例)           │
│                                         │
├─────────────────────────────────────────┤
│  相册标题                    照片数量    │
│  副标题 · 日期 · 地点                    │
└─────────────────────────────────────────┘
```

### 封面图规范

- **比例保持原图**：不要强制裁切成正方形或固定比例，用 `object-fit: contain` 或让容器高度自适应图片比例
- **常见封面比例**：16:9（风景）、3:2（标准照片）、4:3（人像）
- **最大高度限制**：全宽卡片封面最大高度 70vh，半宽卡片最大高度 50vh，防止超长图片占据整个屏幕
- **无圆角**：`border-radius: 0`
- **无阴影**：`box-shadow: none`
- **无边框**：靠背景色自然区分

### 文字信息区

**位置**：封面图下方，左对齐，与图片左边缘对齐
**顶部间距**：封面图与文字之间 20px

**文字层级：**
- 相册标题：H2 字号（28-32px），字重 600，颜色主文字色，Noto Serif SC
- 副标题/日期/地点：Caption 字号（13-14px），颜色辅助文字色，系统字体
  - 格式：`"2024 · 厦门"` 或 `"2024 · 厦门 · 15 张照片"`
  - 用中点 `·` 作为分隔符，不要用斜杠或横杠

**照片数量：**
- 显示在标题右侧或副标题中，格式："12 张照片"
- 颜色弱化文字色

### 悬停交互

- **封面图**：`scale(1.03)`，过渡 0.4s ease
- **标题**：颜色变为点缀色（浅色 `#8B7355`，深色 `#C4A87A`），过渡 0.3s
- **鼠标指针**：`cursor: pointer`
- **整个卡片可点击**：点击后导航到相册详情页

### 点击态（Active）

- 图片瞬间 `scale(0.98)`，给用户明确的点击反馈，持续 0.1s

## 七、相册详情页照片墙设计

### 页面结构

```
┌─────────────────────────────────────────────┐
│  Header                                     │
├─────────────────────────────────────────────┤
│  返回按钮 (BackButton)                       │
├─────────────────────────────────────────────┤
│                                             │
│  相册 Hero 区域                              │
│  标题 + 副标题 + 简介 + 日期/地点            │
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│  照片展览区 (PhotoWall)                      │
│  错落排列的照片，不同尺寸                    │
│                                             │
├─────────────────────────────────────────────┤
│  Footer                                     │
└─────────────────────────────────────────────┘
```

### 相册 Hero 区域

**与首页 Hero 的区别：**
- 不用全屏背景图，用纯色背景（与页面背景一致 `#FAFAF8`）
- 文字居左对齐，不是居中
- 更紧凑，高度 auto，靠 padding 撑开

**具体尺寸：**
- 顶部 padding：120px（桌面），80px（移动）
- 底部 padding：80px（桌面），60px（移动）
- 左右 padding：与页面一致 48px（桌面），24px（移动）

**文字层级：**
- 返回按钮：顶部固定，左侧，Caption 字号，带左箭头图标
- 相册标题：H1 字号（42-48px），字重 600，Noto Serif SC，颜色主文字色
- 副标题：Body 字号（16px），颜色辅助文字色，与标题间距 12px
- 简介：Body 字号（16px），颜色辅助文字色，行高 1.8，最大宽度 640px，与副标题间距 24px
- 日期/地点：Caption 字号（13-14px），颜色弱化文字色，格式 `"2024.06.15 · 厦门·鼓浪屿"`

### PhotoWall 展览式布局

**设计理念：**
照片像挂在画廊墙上的作品，不同尺寸创造视觉节奏。没有固定网格，而是根据每张照片的`layout`字段决定其展示尺寸。

**三种布局尺寸：**

| layout 类型 | 宽度占比 | 适用场景 | 高度处理 |
|-----------|---------|---------|---------|
| **full** | 100%（单列） | 代表作、横图大作 | 高度自适应，max-height: 85vh |
| **half** | 50%（两列并排） | 常规作品、竖图 | 高度自适应，两张并排时取较高者 |
| **large** | 66.67%（2/3 宽度） | 重要作品，比 half 更突出 | 高度自适应，右侧可留白或放小图 |

**布局组合示例：**

```
桌面端（max-width: 1200px，居中，gap: 40px）：

┌─────────────────────────────────────────────┐
│              Photo (full)                    │
│              大海日落全景                    │
│              下方: "日落礁石"                 │
├────────────────────────┬────────────────────┤
│   Photo (half)          │   Photo (half)      │
│   浪花特写              │   礁石纹理          │
│   下方: "浪花"          │   下方: "礁石"       │
├─────────────────────────────────────────────┤
│              Photo (full)                    │
│              人物剪影                        │
│              下方: "海边的人"                 │
├────────────────────────┬────────────────────┤
│   Photo (half)          │   Photo (half)      │
│   船只                  │   海鸥              │
├─────────────────────────────────────────────┤
│   Photo (large, 2/3)    │   Photo (half, 1/3) │
│   重点作品              │   辅助作品          │
└─────────────────────────────────────────────┘
```

**间距规范：**
- 照片之间 gap：40px（桌面），24px（移动）
- 每张照片底部与标题间距：12px
- 照片组之间（不同行）间距：60-80px（桌面），40px（移动）
- 页面左右 padding：48px（桌面），24px（移动）

**照片标题：**
- 位置：照片正下方，左对齐
- 字号：Caption（13-14px）
- 颜色：辅助文字色 `#6B6B6B`（浅色）/ `#9A9A9A`（深色）
- 字体：Noto Serif SC
- 不显示描述：照片墙只显示标题，描述在照片详情页展示

**照片本身规范：**
- `border-radius: 0`（无圆角）
- `box-shadow: none`（无阴影）
- 保持原图比例
- 图片下方可以加极淡的细线分隔（1px，`#E8E4DE`），也可以不加，靠留白分隔

**移动端适配（< 768px）：**
- 所有照片全宽（100%），单列排列
- 照片间距缩减到 24px
- 照片标题字号 12px

### 点击交互

- 点击照片 → 导航到照片详情页
- 悬停照片：`scale(1.02)`，过渡 0.4s ease
- 鼠标指针：`cursor: zoom-in`
- 点击态：瞬间 `scale(0.99)`

## 八、照片详情页 EXIF 面板设计

### 页面结构（桌面端）

```
┌──────────────────────────────────────────────────────────────┐
│  Header                                                      │
├───────────────────────────────┬──────────────────────────────┤
│                               │                              │
│                               │  照片标题                     │
│                               │  "日落礁石"                   │
```

## 九、动效规范

### 动效哲学

**只用三种动效，其他全部禁止。**

动效的目的是让体验更流畅、更自然，而不是吸引注意力。摄影网站的动效应该像翻书页一样安静。

### 允许使用的三种动效

#### 1. 滚动淡入（全局）

**用途：** 页面元素进入视口时的出现动画

**参数：**
- 初始状态：`opacity: 0`, `translateY(20px)`
- 结束状态：`opacity: 1`, `translateY(0)`
- 持续时间：`0.6s`
- 缓动函数：`ease-out`（cubic-bezier: 0.0, 0.0, 0.2, 1）
- 触发条件：元素进入视口（Intersection Observer），提前 50px 触发（`margin: '-50px'`）
- 只触发一次：`once: true`
- 交错延迟（stagger）：多个元素依次出现时，每个延迟 `0.1s`

**使用场景：**
- 首页 AlbumCard 逐个出现
- 相册详情页照片逐个出现
- 关于页面文字段落出现
- 照片详情页信息面板出现

**禁止：** 不要从 scale(0.9) 放大出现，不要从左右两侧滑入，不要带旋转。

#### 2. 悬停微放大（卡片和照片）

**用途：** 鼠标悬停时的微交互

**参数：**
- 效果：`scale(1.03)`（图片）或 `scale(1.02)`（照片墙中的照片）
- 持续时间：`0.4s`
- 缓动函数：`ease`
- 同时伴随：标题颜色变为点缀色（过渡 0.3s）

**使用场景：**
- AlbumCard 悬停
- PhotoWall 中的照片悬停
- 导航链接悬停

**禁止：** 不要上下浮动、不要旋转、不要阴影扩散、不要颜色滤镜。

#### 3. 页面过渡淡入

**用途：** 页面切换或照片详情页加载时的过渡

**参数：**
- 效果：`opacity: 0 → 1`，可选 `scale: 0.98 → 1`
- 持续时间：`0.3s`
- 缓动函数：`ease-out`

**使用场景：**
- 路由切换时页面整体淡入
- 照片详情页加载时照片和信息面板分别淡入（信息面板 delay 0.1s）

### 绝对禁止的动效

| 禁止效果 | 原因 |
|---------|------|
| 弹跳（Bounce） | 太活泼，破坏沉静感 |
| 旋转 | 与摄影主题无关，干扰视觉 |
| 打字机效果 | 慢且烦躁 |
| 粒子效果 | 过度设计，抢照片焦点 |
| 视差滚动 | 在照片网站上容易让人头晕 |
| 3D 翻转 | 像 2010 年的幻灯片 |
| 自动轮播 | 用户无法掌控，破坏阅读节奏 |
| 颜色渐变动画 | 背景色跳动不稳定 |
| 震动/抖动 | 让人不安 |
| 加载动画（复杂） | 用简单的淡入即可，不要转圈 |

### 时间参数速查表

| 动效类型 | 持续时间 | 缓动 | 延迟 |
|---------|---------|------|------|
| 滚动淡入 | 0.6s | ease-out | 0（单个），+0.1s stagger（多个） |
| 悬停放大 | 0.4s | ease | 0 |
| 页面过渡 | 0.3s | ease-out | 0 |
| 标题加载 | 0.8s | ease-out | 0.3s |
| Hero 背景缩放 | 8s | ease-out | 0 |
| 颜色过渡（悬停） | 0.3s | ease | 0 |

## 十、哪些设计容易让它变成普通相册，应该避免

### ❌ 陷阱 1：九宫格/均匀网格

**问题：** 所有照片一样大、一样间距，像 Instagram 或网盘。

**解决：** 用 `full` / `half` / `large` 三种尺寸错落排列，创造画廊的"策展感"。

### ❌ 陷阱 2：正方形裁切

**问题：** 强制所有照片 1:1 正方形，破坏摄影师原本的构图。

**解决：** 保持照片原图比例（3:2、2:3、16:9、4:3），让容器自适应图片高度。

### ❌ 陷阱 3：圆角

**问题：** `border-radius: 8px` 或更高，让照片像 App 图标。

**解决：** 照片边缘全部直角（`border-radius: 0`），干净利落，像画廊装裱。

### ❌ 陷阱 4：阴影

**问题：** `box-shadow` 让照片像浮动的卡片，增加视觉噪音。

**解决：** 完全不用阴影，靠留白和背景色区分层次。

### ❌ 陷阱 5：边框

**问题：** 给照片加白色边框或彩色边框，像老式相框或 PPT。

**解决：** 照片直接放在背景色上，没有任何边框。

### ❌ 陷阱 6：彩色背景

**问题：** 不同区块用不同背景色（灰、白、浅蓝交替），破坏整体感。

**解决：** 全站统一背景色（浅色 `#FAFAF8`，深色 `#1A1A1A`），靠留白区分区块。

### ❌ 陷阱 7：信息堆砌

**问题：** 照片下方显示标题 + 描述 + EXIF + 日期 + 地点 + 标签，信息过载。

**解决：** 照片墙只显示标题，其他信息全部放在照片详情页。

### ❌ 陷阱 8：花哨动效

**问题：** 滚动时照片飞入、3D 翻转、粒子效果，让人分心。

**解决：** 只用淡入和微放大两种动效，保持克制。

### ❌ 陷阱 9：无衬线体标题

**问题：** 全站用系统无衬线体，像企业官网或 SaaS 产品。

**解决：** 标题必须用衬线体（Playfair Display + Noto Serif SC），传递文艺感。

### ❌ 陷阱 10：按钮过多

**问题：** "查看详情"、"下载"、"分享"、"收藏"一堆按钮，像电商平台。

**解决：** 整个网站几乎没有按钮。导航用文字链接，返回用箭头，照片可点击但不需要按钮。

### ❌ 陷阱 11：导航复杂

**问题：** 面包屑、侧边栏、下拉菜单、标签云，像门户网站。

**解决：** 导航极简：标题（回首页）+ Albums + About + 主题切换。仅此而已。

### ❌ 陷阱 12：照片尺寸不统一但无序

**问题：** 虽然照片大小不同，但排列毫无规律，像瀑布流瀑布出错了。

**解决：** 按 `full` → `half+half` → `full` → `half+half` 的节奏排列，有韵律感。

## 十一、给 Codex 执行前端样式时的具体要求

### 1. Tailwind CSS 配置要求

**在 `tailwind.config.ts` 中扩展：**

- `darkMode: 'class'` —— 用 class 策略切换深色模式
- 自定义颜色：把上面配色表中的 7 个色值定义成 Tailwind 颜色名
  - 建议命名：`bg-primary`（背景）、`text-primary`（主文字）、`text-secondary`（辅助文字）、`text-muted`（弱化文字）、`accent`（点缀色）、`border-subtle`（边框）
- 自定义字体：
  - `font-serif: ['"Playfair Display"', '"Noto Serif SC"', 'serif']`
  - `font-sans: ['-apple-system', 'BlinkMacSystemFont', '"Segoe UI"', '"PingFang SC"', '"Microsoft YaHei"', 'sans-serif']`
  - `font-mono: ['"JetBrains Mono"', 'monospace']`
- 自定义间距：在标准 spacing 基础上确保有 `120px` 这种大留白值

### 2. CSS 变量定义（`global.css`）

用 CSS 变量定义颜色，通过 `html.dark` 切换：

```css
:root {
  --bg-primary: #FAFAF8;
  --bg-secondary: #F2F0EC;
  --text-primary: #1A1A1A;
  --text-secondary: #6B6B6B;
  --text-muted: #A0A0A0;
  --accent: #8B7355;
  --border: #E8E4DE;
}

html.dark {
  --bg-primary: #1A1A1A;
  --bg-secondary: #242424;
  --text-primary: #F0EDE8;
  --text-secondary: #9A9A9A;
  --text-muted: #666666;
  --accent: #C4A87A;
  --border: #333333;
}
```

所有颜色使用 `bg-[var(--bg-primary)]` 或 Tailwind 自定义颜色，不要用硬编码色值。

### 3. 字体加载

在 `global.css` 顶部引入 Google Fonts：

```css
@import url('https://fonts.googleapis.com/css2?family=Playfair+Display:ital,wght@0,400;0,600;0,700;1,400&family=Noto+Serif+SC:wght@400;600;700&family=JetBrains+Mono:wght@400;500&display=swap');
```

### 4. 布局实现细节

**AlbumGrid（首页错落布局）：**
- 用 CSS Grid：`grid-template-columns: repeat(2, 1fr)`
- 全宽卡片：`col-span-2`
- 半宽卡片：`col-span-1`
- `gap: 40px`（桌面），`gap: 24px`（移动）
- 最大宽度：`max-w-[1200px]`，居中 `mx-auto`

**PhotoWall（展览式照片墙）：**
- 同样用 CSS Grid：`grid-template-columns: repeat(6, 1fr)`（更细的粒度）
- `full`：`col-span-6`
- `half`：`col-span-3`
- `large`：`col-span-4`
- 如果 `large` 和 `half` 并排，`large` 占 4 列，`half` 占 2 列
- `gap: 40px`（桌面），`gap: 24px`（移动）
- 每张照片用 `<figure>` 包裹，`<figcaption>` 放标题

**照片详情页（桌面端左右布局）：**
- 外层 Flex：`flex flex-row`
- 左侧图片：`w-[60%]` 或 `w-[65%]`
- 右侧信息：`w-[35%]` 或 `w-[40%]`，加 `pl-12`（48px 留白）
- 移动端：`flex flex-col`，上图下信息

### 5. 图片处理规范

- 所有照片：`<img>` 标签，`loading="lazy"`（除了照片详情页首屏用 `eager`）
- 不要用 `<div>` 背景图：用 `<img>` 保持语义化和可访问性
- 图片容器：`overflow-hidden`（配合悬停放大时裁切）
- 不要给 `<img>` 加 `rounded-*`，保持直角
- 图片下方标题用 `<figcaption>`，样式见第七部分

### 6. 组件级样式要求

**Header：**
- 固定定位：`fixed top-0 left-0 right-0 z-50`
- 高度：`h-16`（64px）
- 背景：`bg-[var(--bg-primary)]/80 backdrop-blur-sm`（半透明 + 毛玻璃）
- 底部边框：`border-b border-[var(--border)]`
- 内容：`flex justify-between items-center`，左右 padding 与页面一致

**Footer：**
- 极简：只有一行小字 "© 2024 T&K Photo · Shot with love"
- 文字居中，`text-caption`，`text-muted`
- 上下 padding：`py-12`

**BackButton：**
- 位置：页面左上角，在 Header 下方（`mt-24` 给 Header 让位）
- 样式：左箭头图标 + "返回" 文字，`text-caption`，`text-secondary`
- 悬停：`text-accent`

**ThemeToggle：**
- 位置：Header 右侧
- 图标：太阳/月亮，20x20px
- 切换时：图标旋转 180° + 淡入（0.3s）

### 7. 响应式断点

- 桌面：`>= 1024px`（lg）
- 平板：`>= 768px`（md）
- 移动：`< 768px`（默认）

关键响应式变化：
- AlbumGrid / PhotoWall：md 以下全部单列
- 照片详情页：md 以下改为上图下信息（`flex-col`）
- Hero 标题：md 以下缩小字号
- 页面 padding：`px-12`（桌面 48px）→ `px-6`（移动 24px）

### 8. 可访问性要求

- 所有图片加 `alt` 文字（照片标题或描述）
- 颜色对比度满足 WCAG AA（已验证的配色方案）
- 键盘导航：照片详情页支持左右箭头和 ESC
- 减少动效：`@media (prefers-reduced-motion: reduce)` 时禁用所有 Framer Motion 动画

### 9. 性能要求

- 图片懒加载（首屏外全部 `loading="lazy"`）
- 字体 `display=swap`（已包含在 Google Fonts URL 中）
- 不要引入整个图标库，只用 `lucide-react` 的按需导入

### 10. 检查清单（实现后对照）

- [ ] 全站没有 `border-radius` 应用在照片上
- [ ] 全站没有 `box-shadow`
- [ ] 照片保持原图比例，没有被强制裁切
- [ ] 标题全部使用衬线体（Playfair Display / Noto Serif SC）
- [ ] EXIF 参数使用 JetBrains Mono
- [ ] 三种动效之外没有其他动画
- [ ] 深色/浅色模式切换正常工作
- [ ] 移动端布局正确，没有横向滚动条
- [ ] 所有悬停态有 `transition` 过渡
- [ ] 页面留白充足，不拥挤

| EXIF 字体 | 等宽字体 | 档案感、精确感 |
| 动效数量 | 3 种 | 克制 |
| 按钮数量 | 极少 | 非工具类产品 |
| 照片排列 | 错落 Grid | 策展感 |
| 信息展示 | 分层（列表页→详情页）| 不堆砌 |

> 这份设计的核心是让网站"消失"——用户进入后，注意力完全在照片上，网站本身不抢戏。留白、衬线体、直角照片、克制动效，这些都是为了让照片成为主角。
